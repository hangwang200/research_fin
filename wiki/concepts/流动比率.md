# 💧 流动比率 · Current Ratio

> **一句话定义**：[[流动比率]] = 流动资产 / 流动负债，衡量企业**未来 12 个月内偿还短期债务**的能力，是 [[Templates/Company-Analysis-Playbook]] §3 中最基础的「**短期偿债**」红线。

---

## 📐 公式

```
Current Ratio = Current Assets / Current Liabilities

其中：
  Current Assets    = 现金 + 短期投资 + 应收账款 + 存货 + 其他流动资产
  Current Liabilities = 应付账款 + 短期借款 + 一年内到期的长期借款 
                      + 应付薪酬 + 应付税款 + 合同负债 / 预收账款
```

> ⚠ [[Templates/Company-Analysis-Playbook]] §3 强制：剔除「[[合同负债]]」和「[[预收账款]]」后再算，更接近真实「短期偿付义务」。

## 🎯 为什么它重要？

| 维度 | 答案 |
|---|---|
| 防止什么风险 | [[流动性危机]] / [[Working Capital Crunch]] / [[挤兑]]（针对金融机构）|
| 谁最关心 | 银行（授信前置条件）/ 评级机构 / 价值投资者 |
| 配套指标 | [[Quick Ratio]]（速动比率，去掉存货）/ [[Cash Ratio]]（现金比率，仅留现金）/ [[Cash Conversion Cycle]] |

---

## 🚨 红线（[[Templates/Company-Analysis-Playbook]] §3）

| 流动比率 | 评级 | 行业适用 |
|---|---|---|
| **> 2.0** | ⭐⭐⭐⭐⭐ Strong | 制造业 / 零售业 |
| **1.0 - 2.0** | ⭐⭐⭐⭐ Healthy ✅ | 多数行业（基础门槛）|
| **0.7 - 1.0** | ⭐⭐ Warning ⚠ | 仅平台型/SaaS 可接受（[[预收账款]]多）|
| **< 0.7** | 🚨 Danger | [[流动性风险]] |

> ⚠ 太高（> 3.0）也未必好 —— 可能是 [[资金闲置]] / [[低效经营]] / [[现金堆积]] 信号

## ⚠️ 行业差异（不能一刀切）

| 行业 | 健康流动比率 | 解读 |
|---|---|---|
| **[[制造业]]** | 1.5 - 2.5 | 存货周期长，需备额外缓冲 |
| **[[零售业]]** | 1.2 - 2.0 | 存货+应收较多 |
| **[[平台型]]** / [[OTA]] | 1.0 - 1.5 | [[预收账款]] / [[合同负债]] 占大头 |
| **[[SaaS]]** | 0.8 - 1.5 | [[递延收入]] 是主要流动负债 |
| **[[银行]]** / [[保险]]** | 不适用 | 看 [[资本充足率]] / [[偿付能力]] 替代 |
| **[[公用事业]]** | 0.5 - 1.0 | 稳定现金流不需高比率 |

## 🔍 配套指标（精度递增）

```
[[Quick Ratio]]（速动比率）   = (流动资产 - 存货) / 流动负债          阈值 > 1
[[Cash Ratio]]（现金比率）   = (现金 + 短期投资) / 流动负债         阈值 > 0.3
[[Cash Conversion Cycle]] = DSO + DIO - DPO                         阈值 < 60 天
```

> ⭐ 价值投资者常组合使用：流动比率 > 1.5 + 速动比率 > 1 + Cash Conversion Cycle 趋势改善 = 真实健康。

## 🚩 [[Window Dressing]] 警惕（季末粉饰）

公司可能在季末刻意操作让流动比率好看：
1. 12 月集中收回 [[应收账款]]
2. 推迟支付 [[应付账款]]（[[Stretching AP]]）
3. 加速短期投资变现
4. 通过 [[Repo]] / [[商业票据]] 短期融资美化

> 应对：用**季度均值**或**4 季度移动平均**替代年末快照

## 🔗 与其他概念的关系

- **[[Tangible Net Worth]]**：长期偿债指标，对照短期偿债的流动比率
- **[[Quick Ratio]]**：流动比率的「严格版」（去掉存货）
- **[[Working Capital]]**：= 流动资产 - 流动负债（绝对额版本）
- **[[Cash Conversion Cycle]]**：从经营周期视角衡量流动性效率
- **[[Altman Z-Score]]**：综合破产预警模型，流动比率是输入之一

## ⚠️ 实战陷阱

1. **[[预收账款]] / [[合同负债]] 占大头**：[[OTA]] / [[SaaS]] 的「短期偿付」实际上不是「现金支付」而是「服务交付」，比率严苛意义不大 → [[TCOM]] / [[Booking]] / [[Salesforce]] 都属此类
2. **[[应付商业票据]] vs [[应付账款]]**：[[Apple]] / [[Costco]] 的强势让供应商免息融资，导致流动比率长期 < 1，但反而是商业模式好的体现
3. **[[受限现金]]** 应剔除：[[VIE]] 资产、[[履约保证金]]、跨境资金池受限部分
4. **[[存货跌价]]** 风险：科技 / 时尚业存货账面价值可能虚高
5. **[[长期借款]] 1 年内到期部分**：必须重分类到流动负债，否则流动比率被高估
